# product-owner

Product ownership workflow for turning goals, stakeholder needs, rough ideas, or Jira tickets into
clear, testable, implementation-ready work.

See [SKILL.md](./SKILL.md) for the full workflow.

## What it does

- Clarifies product goals, user value, stakeholders, scope, assumptions, and dependencies.
- Writes Jira-ready stories, tasks, defects, and acceptance criteria.
- Calls out UX considerations, edge cases, non-functional requirements, and handoff notes.
- Collaborates with [`software-engineer`](../software-engineer/),
  [`manual-tester`](../manual-tester/), and [`test-automation-engineer`](../test-automation-engineer/)
  without duplicating their workflows.
