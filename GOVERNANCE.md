# Governance

`agent-skills` is currently maintained by a single maintainer.

## Maintainer Role

The maintainer is responsible for:

- Setting project direction and release timing.
- Reviewing issues and pull requests as capacity allows.
- Deciding when a proposed skill belongs as a top-level role, a nested skill, a reference document, or outside the project scope.
- Protecting the repository's portability, public-benefit mission, and documentation quality.

## Decision Making

Most decisions are made through issues and pull requests. The maintainer will favor changes that are:

- Broadly reusable across organizations and stacks.
- Simple enough for contributors to understand and maintain.
- Grounded in real engineering workflows.
- Compatible with the existing skill structure.

Large changes should start as an issue before a pull request. Examples include new top-level roles, required environment variable changes, folder layout changes, or workflow changes that would affect existing users.

## Future Maintainers

Additional maintainers may be invited if the project has sustained external contribution and shared maintenance work becomes necessary. Until then, the project should not pretend to have committees, working groups, or formal voting processes.

## Community Expectations

Be direct, kind, and practical. Critique ideas and changes, not people. Contributions that are hostile, spammy, discriminatory, or unrelated to the project may be closed or removed.
