# Governance

Solo project. One maintainer. No committees, no working groups, no formal voting.

The maintainer decides direction, release timing, scope boundaries, what gets merged, and what doesn't — without owing anyone an explanation. This is intentional: it keeps the project small enough for one person to maintain alongside their day job.

If multiple people end up doing sustained, high-quality work on the project over time, this file will be revisited. Until then, please don't read more structure into the project than is actually here.

For expectations on contributions, see [CONTRIBUTING.md](./CONTRIBUTING.md).
